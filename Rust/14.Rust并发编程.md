# Rust 并发编程

## 线程

- 线程不同于进程（process）的地方是线程是程序以内的概念，程序往往是在一个进程中执行的。

- Rust 在语言本身就设计了包括所有权机制在内的手段来尽可能地把最常见的错误消灭在编译阶段，这一点其他语言不具备

```rust
use std::thread;
use std::time::Duration;

fn spawn_function() {
    for i in 0..5 {
        println!("spawned thread print {}", i);
        thread::sleep(Duration::from_millis(1));
    }
}

fn main() {
    thread::spawn(spawn_function);

    for i in 0..3 {
        println!("main thread print {}", i);
        thread::sleep(Duration::from_millis(1));
    }
}
```

- std::thread::spawn 函数的参数是一个无参函数

```rust
use std::thread;
use std::time::Duration;

fn main() {
    // 可以使用闭包（closures）来传递函数作为参数
    // |参数1, 参数2, ...| -> 返回值类型 {
    //     函数体
    // }
    thread::spawn(|| {
        for i in 0..5 {
            println!("spawned thread print {}", i);
            thread::sleep(Duration::from_millis(1));
        }
    });

    for i in 0..3 {
        println!("main thread print {}", i);
        thread::sleep(Duration::from_millis(1));
    }
}
```

## join 方法

- join 方法可以使子线程运行结束后再停止运行程序。

```rust
use std::thread;
use std::time::Duration;

fn main() {
    let handle = thread::spawn(|| {
        for i in 0..5 {
            println!("spawned thread print {}", i);
            thread::sleep(Duration::from_millis(1));
        }
    });

    for i in 0..3 {
        println!("main thread print {}", i);
        thread::sleep(Duration::from_millis(1));
    }

    handle.join().unwrap();
}
```

## move 强制所有权迁移

```rust
use std::thread;

fn main() {
    let s = "hello";
    
    let handle = thread::spawn(move || {
        // 子线程中尝试使用当前函数的资源, 使用闭包的 move 关键字来处理
        println!("{}", s);
    });

    handle.join().unwrap();
}
```

## 消息传递

- Rust 中一个实现消息传递并发的主要工具是通道（channel），通道有两部分组成，一个发送者（transmitter）和一个接收者（receiver）。

```rust
use std::thread;
use std::sync::mpsc;

fn main() {
    let (tx, rx) = mpsc::channel();

    thread::spawn(move || {
        let val = String::from("hi");
        tx.send(val).unwrap();
    });

    let received = rx.recv().unwrap();
    println!("Got: {}", received);
}
```
