# C# <ruby>基本文法<rt>きほんぶんぽう</rt></ruby>

---

## 導入（どうにゅう）

みなさん、こんにちは。  
今日のテーマは **C# の<ruby>基本文法<rt>きほんぶんぽう</rt></ruby>** です。  

C# は <ruby>面向对象<rt>めんこうたいしょう</rt></ruby>、つまり **オブジェクト<ruby>指向<rt>しこう</rt></ruby>** のプログラミング言語です。  
プログラムは「オブジェクト」という<ruby>単位<rt>たんい</rt></ruby>で構成されます。  
同じ種類のオブジェクトは同じ **クラス（class）** にまとめられます。  
この「クラス」が C# における最も重要な<ruby>概念<rt>がいねん</rt></ruby>のひとつです。  

---

## using キーワード

C# のプログラムでは、最初に **using** キーワードを使います。  
これは標準機能を使うための宣言で、C# の便利な機能やライブラリを使うときに必要です。  

---

## class キーワード

プログラムの部品である「クラス」を作るときは、**class** キーワードを使います。  
クラスはオブジェクトの設計図のようなもので、変数や処理のまとまりを定義します。  

---

## コメントについて

コメントは、コードの説明を書くためのものです。  
コンパイラはコメントを無視するので、プログラムの動作には影響しません。  

---

## トップレベルステートメント（Top-Level Statements）

C# 9.0 からは、**トップレベルステートメント** という書き方が追加されました。  
これを使うと、クラスや Main メソッドを作らなくても、ファイルの先頭に直接プログラムを書くことができます。  

---

## まとめ

今日学んだことを整理します。  

1. **C# はオブジェクト指向言語**で、プログラムはクラスから成り立っている  
2. **using** は「ライブラリを借りる合図」  
3. **class** は「設計図」、オブジェクトは「製品」  
4. **コメント** は「メモ」  
5. **トップレベルステートメント** は「シンプルに直接書ける方法」  

---

## 日常たとえ

- **using** → 図書館で本を借りるときの「利用カード」  
- **class** → レシピ（設計図）から料理を作る  
- **オブジェクト** → 設計図から実際にできたスマホや自転車  
- **コメント** → 付箋メモや料理本の余白のメモ  
- **トップレベルステートメント** → メモ帳に直接「買い物リスト」を書く  

---

## 終わりに

C# の基本文法は、プログラムの第一歩です。  
ただ「形を覚える」だけでなく、実際に **コードを書きながら理解すること** が大切です。  
日常のイメージと結びつけて覚えると、自然に身につきます。
